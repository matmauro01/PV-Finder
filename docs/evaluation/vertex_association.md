# Evaluation — Vertex Association

Evaluation of the GNN TTVA model on reconstructed primary vertices.

## Classification

Each reconstructed PV is classified as one of:

| Category | Definition |
|----------|-----------|
| **Clean** | Dominant truth PV contributes >= 70% of matched tracks |
| **Merged** | Dominant truth PV contributes < 70% (tracks from multiple truth PVs) |
| **Split** | Another reco PV already claimed the same truth PV with higher sum(pT²) |
| **Fake** | No matched tracks have truth associations |

## Evaluation Methods

- **MaxScore**: For each track, keep only the top-1 highest-scoring PV edge (if above threshold). Each track assigned to at most one PV.
- **Threshold**: Accept all edges with score above threshold. A track can be associated to multiple PVs.

## Usage

Two steps — build inference graphs from PVF histograms, then evaluate:

```bash
# 1. PVF histograms (.npy / pickle / .h5) → inference graphs
python -m gnn.data.pvf_to_graphs \
    -r /path/to/pvf_histograms.p \
    -f /share/lazy/qibinlei/recoTracks_incamvfassoc.h5 \
    -i configs/qibin_test_main_indices_v2.p \
    -o outputs/<date>/inference_graphs.pt
    # peak defaults: -t 1e-2 --integral-threshold 0.2 --min-width 3 (baseline)

# 2. Evaluate
python -m gnn.evaluation.evaluate_ttva \
    -r outputs/<date>/inference_graphs.pt \
    -f /share/lazy/qibinlei/recoTracks_incamvfassoc.h5 \
    -i configs/qibin_test_main_indices_v2.p \
    -w model_weights/gnn_ttva_epoch100.pyt \
    -e MaxScore -t 0.5 -d 0 \
    -o outputs/<date>/
```

Results are saved as `.npy` files: per-event rows
`[clean, merged, split, fake, n_reco, n_truth]` plus per-PV info dicts.

## Results (2026-07-12, GNN e100, MaxScore t=0.5, 2,550 test events)

Truth PVs (nTrk≥2): 72,189. Full details in
`outputs/07_12_2026_ttva_reproduction/reproduction_summary.md`.

| PV source + association | Reco PVs | Clean | Merged | Split | Fake |
|---|---|---|---|---|---|
| PVF e400 peaks + GNN (Nov 2025 baseline) | 66,472 | 76.04% | 16.49% | 7.05% | 0.42% |
| PVF e400 peaks + GNN (graphs regenerated by this repo) | 66,472 | 76.04% | 16.49% | 7.05% | 0.42% |
| Qi Bin UNet-85 histograms + GNN | 65,671 | 76.84% | 16.53% | 6.30% | 0.33% |
| Ground-truth vertices + GNN (upper bound) | 80,498 | 76.74% | 11.67% | 5.59% | 6.00% |
| **AMVF (its own finding + association)** | 57,329 | 76.69% | 20.05% | 2.51% | 0.75% |

**AMVF comparison** (`gnn.evaluation.evaluate_amvf_ttva`, identical
`classify_assignments` core, same events): AMVF's clean *rate* matches ours
(76.7% vs 76.0%), but it reconstructs 16% fewer vertices (57,329 vs 66,472 —
79.4% vs 92.1% of truth). Per truth PV, clean-vertex efficiency:
**PVF+GNN 70.0% vs AMVF 60.9%**. AMVF merges more (20.1% vs 16.5%, consistent
with its ~2× worse σ_vtx-vtx); PVF+GNN splits more (7.1% vs 2.5%, the price of
conjoined-peak splitting).

**Verification level:** rows 1–2 are bit-identical to the Nov 2025 saved
baseline, per event, and the regenerated graphs are tensor-identical to the
Nov 2025 graph file — the full pipeline (histogram → peak finding → graph →
GNN → classification) is validated. The internal note quotes ~70–75% clean /
5–10% fake on PVF vertices and ~77–78% clean on ground truth (preliminary,
pre-dating the `integral_threshold=0.2` fix); our numbers sit at or above
that envelope and match its ground-truth baseline.

Peak finding: `threshold=1e-2`, `integral_threshold=0.2`, `min_width=3`.
PVF model behind e400: `e2e_mlpHist50_e2e400_1latent_mse_phase2_epoch_400.pyt`
(in `~/codice/atlas_pvfinder/model_weights/e2e_attempt4_mlp_hist/`).
Ground-truth graphs include nTrk<2 truth vertices, hence reco > truth and the
6% fake rate in that row.

gnn.evaluation.evaluate_amvf_ttva` with data arguments hangs at 100% CPU
